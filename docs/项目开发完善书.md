## 一、项目概述

### 1.1 项目简介

BlueprintDigitalNexus 是一个基于 Spring Boot 3 的企业级仓库管理系统（WMS），采用 RBAC 权限控制模型，支持用户认证、角色管理、菜单权限、商品管理、仓库管理、库存管理、出入库业务、操作日志、库存变动日志、首页看板等核心功能。

### 项目整体结构

```
系统基础层
├── 员工管理（用户）
├── 角色管理（权限核心）
├── 菜单管理（路由控制）
├── 系统配置（sys_config key-value）

业务层
├── 商品管理
├── 分类管理
├── 仓库管理
├── 入库管理（完整状态流转）
├── 出库管理（完整状态流转）

辅助层
├── 库存管理（三种调整模式 + 乐观锁）
├── 库存变动日志（wms_stock_log 自动记录）
├── 操作日志（@OperLog + AOP 自动记录）
├── 首页看板（Dashboard 聚合查询 + Redis 缓存）
```

### 2.17 RBAC 权限缓存模块 ✅（新增）

| 功能 | 状态 | 说明 |
|------|------|------|
| JWT 精简 | ✅ | 只存 userId + username，不再存 roleKeys |
| 实时查角色 | ✅ | AdminJwtInterceptor 从数据库/Redis 实时查询 |
| 权限缓存 | ✅ | @Cacheable("WMS:PERMISSION")，TTL 2小时 |
| 缓存清除 | ✅ | 改角色/改菜单时 @CacheEvict 主动清缓存 |
| AOP 自调用 | ✅ | @Lazy 注入 self 代理，解决缓存自调用失效 |
| 用户权限接口 | ✅ | GET /admin/user/permissions，返回 roles + permissions + 菜单树 |
| 菜单树过滤 | ✅ | 过滤禁用角色（status != 1），只返回有权限的菜单 |

**核心设计说明：**

```java
// 1. JWT 只存身份信息，不存角色
// JwtUtil 生成 token 时只写入 userId + username

// 2. AdminJwtInterceptor 实时查角色
// 解析 JWT 后，通过 PermissionService.getUserRoles(userId) 从 Redis/DB 获取角色

// 3. 权限缓存
@Cacheable(value = "WMS:PERMISSION", key = "'perm:' + #userId")
public UserPermissionVO getUserPermissions(Long userId) { ... }

// 4. 缓存清除（改角色/改菜单时）
@CacheEvict(value = "WMS:PERMISSION", allEntries = true)
public void bindMenu(...) { ... }

// 5. 解决 AOP 自调用
@Lazy
@Autowired
private PermissionServiceImpl self;

// 通过 self 调用确保走代理
return self.getUserPermissions(userId);
```

**测试结果（全部通过）：**

| 测试场景 | 结果 |
|---------|------|
| 缓存命中 | ✅ 连续调两次，第二次走 Redis 无 SQL |
| 多角色合并 | ✅ roles=[SUPER_ADMIN, TEMP_TEST]，31个权限，40个菜单 |
| 清空角色 | ✅ 全部返回空 |
| 禁用角色 | ✅ status=0 的角色被过滤，全返回空 |
| 恢复后 | ✅ 权限和菜单立即恢复 |


## 七、注意事项

### 7.1 权限注解

当前部分 Controller 的权限注解被注释（暂时放开测试）。上线前需要启用：

```java
@RequiresRole({RoleConstant.SUPER_ADMIN, RoleConstant.ADMIN})
@RequiresPermission("system:user:view")
```

### 7.2 事务控制

所有写操作必须加 `@Transactional(rollbackFor = Exception.class)`，特别是涉及多表操作的方法。

### 7.3 乐观锁

库存更新使用 `@Version` 乐观锁，防止并发冲突。更新失败会抛出 `OptimisticLockingFailureException`。

### 7.4 逻辑删除

全局配置 `is_deleted` 字段，`0=未删除，1=已删除`。`@TableLogic` 注解自动处理。

### 7.5 Redis 缓存策略

| 缓存空间 | TTL | 说明 |
|---|---|---|
| WMS:WAREHOUSE | 30天 | 基础数据极少修改 |
| WMS:CATEGORY | 7天 | 低频修改 |
| WMS:PRODUCT | 1天 | 偶尔修改 |
| WMS:PERMISSION | 2小时 | 权限缓存，改角色/菜单时主动清除 |
| WMS:DASHBOARD | 10秒 | 看板数据，配合前端30秒轮询 |
| 默认 | 30分钟 | 未显式配置的缓存空间 |

### 7.6 OSS 配置

OSS 的 `access-key-id` 和 `access-key-secret` 使用环境变量配置：

```yaml
aliyun:
  oss:
    access-key-id: ${OSS_ACCESS_KEY_ID}
    access-key-secret: ${OSS_ACCESS_KEY_SECRET}
```

部署时需设置环境变量 `OSS_ACCESS_KEY_ID` 和 `OSS_ACCESS_KEY_SECRET`。

### 7.7 application.yml 安全

`application.yml` 已加入 `.gitignore`，避免敏感配置泄露。部署时需要手动创建或从模板复制。
