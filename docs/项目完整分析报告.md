## 四、架构分析

### 4.1 分层架构（标准三层）

```
Controller（接收请求、参数校验、日志、@OperLog操作日志）
    ↓
Service（业务逻辑、事务、缓存）
    ↓
Mapper（数据访问，MyBatis-Plus BaseMapper）
```

### 4.2 认证架构

```
请求 → WebConfig 拦截器路径匹配
    → /admin/** → AdminJwtInterceptor → 解析 JWT → UserContext (ThreadLocal)
    → /user/**  → UserJwtInterceptor  → 解析 JWT → UserContext
    → /ai/**    → 无认证（直接放行）
```

### 4.3 AOP 切面架构

```
@RequiresPermission / @RequiresRole  →  PermissionCheckAspect / RoleCheckAspect（权限校验）
@OperLog                              →  OperLogAspect（操作日志记录）
```

### 4.4 缓存架构

```
CacheConfig 配置 RedisCacheManager
    → 仓库 TTL 30天
    → 分类 TTL 7天
    → 商品 TTL 1天
    → 默认 TTL 30分钟

Service 层用 @Cacheable / @CacheEvict 注解
    → @Lazy 自注入解决内部方法调用缓存失效
```

### 4.5 编码生成架构

```
RedisIdGenerator（Redis INCR 优先）
    ↓ 失败时
SequenceSyncService（数据库 SELECT MAX 兜底）
    ↓
CodeGenerator（拼接前缀 + 日期 + 补零序号）
```

## 五、做得好的地方

### 5.1 工程规范

- **统一响应体** Result<T>，所有接口返回格式一致
- **全局异常处理** GlobalExceptionHandler 覆盖了 10+ 种异常类型
- **MapStruct** 编译期对象映射，比 BeanUtils 更安全更快
- **日志规范** 统一使用 @Slf4j，关键操作都有 log.info
- **注释到位** 每个类、每个方法都有 Javadoc 注释
- **常量提取** 错误信息、状态码全部集中在 Constant 类中

### 5.2 技术亮点

- **乐观锁** @Version 防库存并发冲突
- **Redis + 数据库双重编码生成** 启动同步 + 降级兜底
- **@Lazy 自注入** 解决 Spring 缓存自调用失效的坑
- **BizValidateUtil** 通用校验工具，新增/编辑唯一性校验复用
- **AOP 权限切面** 声明式权限，不侵入业务代码
- **AOP 操作日志** @OperLog 注解一行代码自动记录审计日志

### 5.3 数据库设计

- **逻辑删除** 全局配置 is_deleted，不物理删除
- **自动填充** createTime / updateTime 由 MetaObjectHandler 处理
- **联合唯一约束** 库存表 (warehouse_id, product_id)
- **角色-菜单多对多** 中间表 sys_role_menu

## 六、问题和优化点

### P0 — 已修复 ✅

| 问题 | 状态 |
|------|------|
| 出库模块未实现 | ✅ 已完成全部7个接口 |
| 缺少操作日志 | ✅ @OperLog + AOP 全覆盖 |

| 出库无单元测试 | ✅ 已有参数校验测试 |

### P1 — 待改进

#### 6.1 Spring Security 形同虚设

**现状：** SecurityConfig 中 `permitAll()`，所有请求都放行。
**建议：** 去掉 Security 依赖，只保留 BCrypt（减少开销）。

#### 6.2 AI 对话模块过于简陋

**现状：** 同步 GET 接口，无流式、无记忆、无业务集成。
**计划：** 后续迭代，配合 React 前端一起做。

#### 6.3 缺少多环境配置

**建议：**
```
application.yml          ← 公共配置
application-dev.yml      ← 本地开发（关 SSL，明文密码）
application-prod.yml     ← 生产环境（开 SSL，环境变量）
```

#### 6.4 BeanUtils.copyProperties 和 MapStruct 混用

**现状：** 出库模块用 BeanUtils，其他模块用 MapStruct。
**建议：** 统一用 MapStruct。

#### 6.5 @Autowired 字段注入 vs 构造器注入混用

**建议：** 统一用构造器注入 + @RequiredArgsConstructor（Lombok）。

#### 6.6 库存变动日志（wms_stock_log）未实现

**现状：** 数据库有 wms_stock_log 表，但代码中没有写入逻辑。
**建议：** 在 adjustInventory 中记录每次库存变动。

### P2 — 锦上添花

| 问题 | 建议 |
|------|------|
| 缺少接口限流 | Guava RateLimiter 或 Redis + Lua |
| Knife4j 生产环境未关闭 | @Profile("!prod") |
| 缺少 Docker 部署配置 | Dockerfile + docker-compose.yml |
| SQL 初始化脚本不规范 | Flyway 管理数据库版本 |
